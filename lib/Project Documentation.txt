# U-Teen Project Documentation

## Overview
U-Teen is a Flutter-based mobile application designed to facilitate food ordering within the UMN (Universitas Multimedia Nusantara) canteen. The app supports both customer and seller roles, allowing customers to browse food items, add them to a cart, place orders, and track their order status, while sellers can manage their products, view orders, and handle financial transactions. The application leverages the Provider package for state management, SharedPreferences for local storage, and various Flutter widgets for a responsive and engaging user interface.

This documentation provides a comprehensive overview of the project’s architecture, file structure, key components, and functionality. It is intended for developers, maintainers, and stakeholders to understand the system and facilitate future development or maintenance.

## Key Components

### Models
Models define the data structures used throughout the application. They include serialization methods (`toMap` and `fromMap`) for persistence with SharedPreferences.

1. **User (`user_model.dart`, `auth_provider.dart`)**
   - Represents a user with properties: `email`, `name`, `userType` (`customer` or `seller`), and an optional `password`.
   - Used for authentication and user-specific data access.
   - Note: `user_model.dart` and `auth_provider.dart` both define a `User` class, which is redundant. The version in `auth_provider.dart` is used in practice.

2. **Balance (`balance_model.dart`)**
   - Represents a seller’s financial balance with properties: `amount` and `history` (list of `BalanceHistory`).
   - `BalanceHistory` tracks transactions (income or withdrawal) with details like `id`, `date`, `type`, `amount`, `description`, `orderId`, and `paymentMethod`.

3. **CartItem (`cart_item.dart`)**
   - Represents an item in the shopping cart with properties: `name`, `price`, `image`, `subtitle`, `sellerEmail`, and `quantity`.
   - Includes equality and hash code overrides for comparison.

4. **FavoriteItem (`favorite_item.dart`)**
   - Represents a favorite food item with properties: `name`, `price`, `image`, and `subtitle`.
   - Supports JSON serialization for persistence.

5. **NotificationModel (`notification_model.dart`)**
   - Represents a notification with properties: `id`, `type` (e.g., `order`, `system`, `promo`), `title`, `message`, `timestamp`, `isRead`, and `payload`.
   - Can be created from an `Order` to notify about order status changes.

6. **Order (`order_model.dart`)**
   - Represents an order with properties: `id`, `orderTime`, `pickupTime`, `items` (list of `OrderItem`), `status`, `paymentMethod`, `merchantName`, `merchantEmail`, `customerName`, `cancellationReason`, `notes`, `completedTime`, `cancelledTime`, `foodRating`, `appRating`, `foodNotes`, and `appNotes`.
   - `OrderItem` represents individual items in an order with properties like `name`, `image`, `subtitle`, `price`, `quantity`, and `sellerEmail`.

7. **PaymentMethod (`payment_method.dart`)**
   - Represents a payment method with properties: `id`, `name`, `iconPath`, `description`, `primaryColor`, `requiresPhoneNumber`, and `supportsTopUp`.

8. **Product (`product_model.dart`)**
   - Represents a food product with properties: `id`, `title`, `subtitle`, `time`, `imgUrl`, `price`, and `sellerEmail`.

### Providers
Providers manage the application’s state using the Provider package, enabling reactive UI updates.

1. **AuthProvider (`auth_provider.dart`)**
   - Manages user authentication state, including login, logout, and initialization from SharedPreferences.
   - Properties: `user`, `isLoading`, `isLoggedIn`, `isSeller`, `isCustomer`, `sellerEmail`, `customerEmail`, `customerName`.

2. **BalanceProvider (`balance_provider.dart`)**
   - Manages seller balances, storing them in SharedPreferences.
   - Functions: Load/save balances, add order income, and process withdrawals.

3. **CartProvider (`cart_provider.dart`)**
   - Manages the shopping cart, including adding/removing items, adjusting quantities, and calculating totals.
   - Provides grouped items by merchant (`subtitle`) for display.

4. **FavoriteProvider (`favorite_provider.dart`)**
   - Manages favorite food items, with persistence in SharedPreferences.
   - Functions: Add/remove favorites, check if an item is favorited, and clear favorites.

5. **FoodProvider (`food_provider.dart`)**
   - Manages food products, loading them from `food_data.dart` and supporting CRUD operations.
   - Functions: Load initial data, add/update/remove products, and filter by seller.

6. **NotificationProvider (`notification_provider.dart`)**
   - Manages notifications, with persistence in SharedPreferences.
   - Functions: Add notifications, mark as read, mark all as read, clear all, and filter by customer.

7. **OrderProvider (`order_provider.dart`)**
   - Manages orders, with persistence in SharedPreferences.
   - Functions: Add orders, update status, submit ratings, create orders from cart, handle withdrawals, and calculate balances/earnings.

8. **RatingsProvider (`ratings_provider.dart`)**
   - Manages ratings for sellers based on completed orders.
   - Functions: Get rated orders, calculate average food rating, and retrieve food notes.

### Screens
Screens represent the main UI views of the application, divided into customer and seller functionalities.

1. **Customer Screens**
   - **FavoriteScreen (`favorite_screen.dart`)**
     - Displays a list of favorite food items.

   - **HomeScreen (`home_screen.dart`)**

   - **MyOrdersScreen (`my_orders_screen.dart`)**

   - **NotificationScreen (`notification_screen.dart`)**

   - **PaymentScreen (`payment_screen.dart`)**

   - **PaymentSuccessScreen (`payment_success_screen.dart`)**

   - **ProfileScreen (`profile_screen.dart`)**

   - **ShoppingCartScreen (`shopping_cart_screen.dart`)**

2. **Seller Screens**

3. **Other Screens**

## Core Functionality

### Authentication
- **Login**: Users log in via `LoginScreen`, providing email, name, and user type (`customer` or `seller`). `AuthProvider` stores credentials in SharedPreferences and sets the user state.
- **Logout**: `LogoutService` displays a confirmation dialog and clears user data via `AuthProvider`.
- **Initialization**: On app start, `AuthProvider` checks SharedPreferences for existing user data to auto-login.

### Food Browsing and Searching
- **Browsing**: `HomeScreen` displays food items from `FoodProvider`, filtered by category. Users can tap items to view details in `DetailBox`.
- **Searching**: `SearchWidget` allows searching food items, with recent searches persisted via `SearchData`.

### Cart Management
- **Adding Items**: Users add items to the cart from `HomeScreen` or `FavoriteScreen`. `CartProvider` manages quantities and groups items by merchant.
- **Modifying Cart**: `ShoppingCartScreen` allows increasing/decreasing quantities, removing items, or clearing the cart.
- **Checkout**: Users proceed to `PaymentScreen` for order placement.

### Order Placement and Tracking
- **Placement**: `PaymentScreen` collects payment method, pickup time, and notes, then creates an `Order` via `OrderProvider`. The cart is cleared, and the user is redirected to `PaymentSuccessScreen`.
- **Tracking**: `MyOrdersScreen` displays ongoing and historical orders. `OrderProvider` supports status updates (pending, processing, ready, completed, cancelled).
- **Ratings**: Customers can submit food and app ratings via `OrderProvider` after order completion.

### Notifications
- **Generation**: `NotificationProvider` creates notifications for order status changes (ready, completed, cancelled) using `NotificationModel.fromOrder`.
- **Display**: `NotificationScreen` shows customer-specific notifications with options to mark as read or view order details.
- **Persistence**: Notifications are stored in SharedPreferences.

### Favorites
- **Management**: `FavoriteProvider` handles adding/removing favorite items, with persistence in SharedPreferences.
- **Display**: `FavoriteScreen` lists favorite items with options to remove or navigate to details.

### Seller Features
- **Product Management**: `FoodProvider` allows sellers to add, update, or remove products.
- **Order Management**: `OrderProvider` provides seller-specific order lists (pending, processing, ready, completed, cancelled).
- **Balance Management**: `BalanceProvider` tracks seller earnings and withdrawals, with transaction history.
- **Ratings**: `RatingsProvider` calculates average food ratings and collects feedback for sellers.

## Setup and Installation
1. **Prerequisites**:
   - Flutter SDK (version compatible with the app, e.g., 3.x.x).
   - Dart (included with Flutter).
   - IDE (e.g., VS Code, Android Studio) with Flutter plugins.
   - Android/iOS emulator or physical device.

2. **Clone the Repository**:
   ```bash
   git clone <repository-url>
   cd u-teen
   ```

3. **Install Dependencies**:
   ```bash
   flutter pub get
   ```

4. **Add Assets**:
   - Ensure the `assets/` directory contains:
     - Lottie animations (`empty_cart.json`, `empty_order.json`, `empty_notification.json`, `success_checkmark.json`).
     - Images (`logo/u.png`, `profile_picture.png`, food images, etc.).
   - Update `pubspec.yaml` to include assets:
     ```yaml
     flutter:
       assets:
         - assets/animation/
         - assets/logo/
         - assets/asset/
         - assets/food/
     ```

5. **Run the App**:
   ```bash
   flutter run
   ```

6. **Build for Release**:
   ```bash
   flutter build apk  # For Android
   flutter build ios  # For iOS
   ```

## Known Issues
1. **Redundant User Class**:
   - Both `user_model.dart` and `auth_provider.dart` define a `User` class. The version in `auth_provider.dart` is used, making `user_model.dart` obsolete.

2. **OrderRepository Underutilized**:
   - `order_repository.dart` defines order creation and retrieval methods but is not integrated into the app. `OrderProvider` handles most order operations.

3. **Hardcoded Profile Data**:
   - `ProfileScreen` uses a hardcoded name (“Javier Matthew”) instead of fetching from `AuthProvider`.

4. **Incomplete Seller Features**:
   - `SellerHomeScreen` is referenced but not provided, indicating incomplete seller functionality.

5. **Potential Navigation Issues**:
   - Frequent use of `pushReplacement` in `WillPopScope` may disrupt the navigation stack. Consider using `push` for non-destructive navigation.

6. **Product Model Syntax Error**:
   - `product_model.dart` contains a syntax error in the constructor (`required String ,` and `String: null`). This needs correction:
     ```dart
     Product({
       required this.id,
       required this.title,
       required this.subtitle,
       required this.time,
       required this.imgUrl,
       required this.price,
       required this.sellerEmail,
     });
     ```

7. **Limited Error Handling**:
   - Some operations (e.g., JSON parsing in providers) catch errors but only log them, without user feedback.
